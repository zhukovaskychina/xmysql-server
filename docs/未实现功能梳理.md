## 四、代码内 TODO/未实现分布（抽样）

以下目录中 **约 95 个 .go 文件** 含有 `TODO`/`FIXME`/`未实现`/`stub` 等标记，需结合具体注释逐项实现或补齐：

- **server/innodb/engine/**：火山执行器、DML、Select/Show、统一执行器、存储适配、索引事务适配等

- **server/innodb/sqlparser/**、**metadata/**、**record/**、**basic/**：解析、元数据、记录格式等
- **server/dispatcher/**、**auth/**、**common/**：系统变量、认证、错误码等

详细列表见：在项目根目录执行  
`rg "TODO|FIXME|未实现|NotImplemented|stub" --type go -l`。

## 五、关键路径（建议实施顺序）

1. **事务与数据正确性**
  ReadView/版本链 → RC/RR 完善 → Undo 回滚/Redo 重放 → 模糊检查点 → Gap/Next-Key 锁  
   （约 25~40 天）
2. **查询正确性与性能**
  子查询执行与优化 → 谓词下推/列裁剪 → 统计信息与选择性 → 连接顺序优化  
   （约 35~45 天）
3. **协议与兼容性**
  预编译语句（COM_STMT_*）→ 认证与 SSL  
   （约 12~18 天）
4. **存储与索引增强**
  B+ 树分裂/合并/二级索引维护 → 表空间/段/页的 P1 增强项  
   （按 STORAGE_ENGINE_FINAL_REPORT 与路线图排期）
