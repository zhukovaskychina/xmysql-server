## 🏗️ 技术架构

### 5层架构设计

```
┌─────────────────────────────────────┐
│      SQL执行层 (Executor)          │
└────────────────┬────────────────────┘
                 │
┌────────────────▼────────────────────┐
│  索引管理层 (IndexManager)         │
│  - 索引生命周期管理                │
│  - 元数据维护                      │
└────────────────┬────────────────────┘
                 │
┌────────────────▼────────────────────┐
│  B+树操作层 (BPlusTreeManager)     │
│  - NodeSplitter (分裂器)          │
│  - NodeMerger (合并器)            │
└────────────────┬────────────────────┘
                 │
┌────────────────▼────────────────────┐
│  缓存管理层 (NodeCache)            │
│  - LRU淘汰策略                     │
│  - 预读优化                        │
└────────────────┬────────────────────┘
                 │
┌────────────────▼────────────────────┐
│  存储管理层 (BufferPoolManager)    │
│  - 页面分配                        │
│  - 脏页管理                        │
└────────────────┬────────────────────┘
                 │
┌────────────────▼────────────────────┐
│  持久化层 (RedoLog/UndoLog)        │
└─────────────────────────────────────┘
```

### 核心组件交互

**插入操作流程** (含分裂)
```
1. Executor.Insert(key, value)
   ↓
2. BPlusTreeManager.Insert(ctx, key, value)
   ↓
3. 定位叶子节点 (O(log N))
   ↓
4. 检查节点容量
   ├─ 未满 → 直接插入
   └─ 满载 → 触发分裂
       ↓
5. NodeSplitter.SplitLeafNode()
   ├─ 分配新页面
   ├─ 分裂数据
   ├─ 更新链表
   └─ 返回middleKey
       ↓
6. NodeSplitter.InsertIntoParent()
   ├─ 父节点未满 → 插入完成
   └─ 父节点满 → 递归分裂
       ↓
7. 标记脏页，异步刷新
```

## 📊 性能指标

### 目标性能矩阵

| 操作 | 当前性能 | 目标性能 | 改进目标 | 优化手段 |
|-----|---------|---------|---------|---------|
| 单点查询 | 0.8ms | < 1ms | 保持 | 缓存优化 |
| 范围扫描(100行) | 8ms | < 5ms | ↑37.5% | 预读+提前终止 |
| 插入(无分裂) | 1.2ms | < 1ms | ↑16.7% | 减少锁持有时间 |
| 插入(含分裂) | 15ms | < 10ms | ↑33.3% | 批量刷新脏页 |
| 删除(含合并) | 18ms | < 10ms | ↑44.4% | 异步合并 |

### 优化策略

**缓存优化**
- 根节点常驻: 避免每次查询加载
- 分层LRU: 内部节点优先级高于叶子节点
- 预读: 顺序访问触发4页预读

**磁盘I/O优化**
- 组提交: 多个事务合并刷新，减少80% fsync
- 异步刷新: 后台线程处理脏页
- 批量写入: 累积脏页批量写入
