## 1. 目的

本文档用于把以下 3 份 P0 文档中的“目标态”映射为“当前仓库已实现状态”：

- `docs/planning/P0_PRODUCTION_TASKS.md`

## 2. 状态定义

- `未实现`：仓库中未发现对应代码、脚本、文档、报告或验收证据
- `部分实现`：有部分代码或文档基础，但距离 P0 验收标准仍有明显缺口
- `已有基础但未达标`：核心能力存在，但缺少自动化、报告、稳定性验证或上线交付物
- `阻塞`：当前存在明确阻塞因素，未解除前无法完成验收

### G-01 Go 版本与测试基线阻塞

- 状态：`阻塞`
- 现状：
  - `go.mod` 声明 `go 1.20`
  - 当前环境 `go version` 为 `go1.16.2`
  - `go test ./server/innodb/engine` 构建失败，报错点为 `errors.Join`
- 影响：
  - 无法完成 A 段和统一发布前检查中的关键测试验收
  - `go test ./...` 当前不可作为 P0 已通过证据
- 建议动作：
  - 先统一开发/CI Go 版本到 1.20+
  - 再重新跑 `server/innodb/engine`、全量回归和多轮稳定性测试
- 优先级：`P0`
- 预计工时：`0.5 人天`
