> 版本：v1.0 | 编制日期：2025-02-09  
> **审核**：见 [开发计划-审核意见.md](./开发计划-审核意见.md)，修订建议已列出，执行前建议按高/中优先级调整。

## 一、计划概要

| 项目 | 说明 |
|------|------|
| **目标** | 将项目完成度从约 85% 提升至 94%，补齐 P0 功能并完成主要 P1 任务 |
| **周期** | 16 周（约 4 个月） |
| **总任务** | 114 项（P0: 37，P1: 67，P2: 10） |
| **预估工作量** | 203～272 人天 |
| **建议人力** | 2～3 名全职开发，可并行多条关键路径 |

## 二、阶段划分与里程碑

| 阶段 | 周次 | 主题 | 里程碑 |
|------|------|------|--------|
| **第一阶段** | 第 1～4 周 | P0 核心突破 | 事务/日志/索引/优化器基础可用，ACID 基本保障 |
| **第二阶段** | 第 5～8 周 | P0 收尾 + P1 启动 | 全部 P0 完成，系统基本可投产 |
| **第三阶段** | 第 9～12 周 | P1 重要功能 | 性能与功能显著提升 |
| **第四阶段** | 第 13～16 周 | P1 收尾 + P2 增强 | 完成度约 94%，达到生产级目标 |

## 三、第一阶段：P0 核心突破（第 1～4 周）

**目标**：完成最关键的 P0 任务，打通事务正确性与数据可靠性两条关键路径。

| 任务 ID | 任务名称 | 模块 | 预估(天) | 产出/验收 |

**周交付**：表达式优化与 MVCC 基础就绪。

### 3.2 第 2 周

| 任务 ID | 任务名称 | 模块 | 预估(天) | 产出/验收 |
|---------|----------|------|----------|------------|
| OPT-011 | 完善谓词下推规则 | 查询优化器 | 4～5 | 可下推谓词识别并应用 |

| TXN-003 | 可见性判断 | 事务管理 | 4～5 | 基于 ReadView 的可见性正确 |

**周交付**：谓词下推与版本链可用，为 RR 打基础。

**验收补充**：`server/innodb/storage/store/mvcc/version_chain_test.go` 已覆盖 TXN-002 版本链创建与遍历、TXN-003 基于 ReadView 的可见版本查找（含跳过删除标记）；`read_view_test.go` 与 `integration_test.go` 覆盖可见性规则与 RC/RR 场景。

### 3.3 第 3 周

| 任务 ID | 任务名称 | 模块 | 预估(天) | 产出/验收 |
|---------|----------|------|----------|------------|

**周交付**：B+ 树可支撑大量写入，Redo 恢复路径打通。

**验收补充**：LOG-001 已通过 `TestCrashRecoveryRedoRestoresData`（完整 Recover + 仅存储重放）验证“崩溃后 Redo 重放可恢复数据”；IDX-001/002 见 `btree_split.go`、`btree_merge.go` 及对应单测。

### 3.4 第 4 周

## 八、风险与应对

| 风险 | 等级 | 应对 |
|------|------|------|
| OPT-018 连接顺序优化复杂 | 高 | 先实现贪心，再考虑 DP；控制表数上限 |
| IDX-001/002 B+ 树分裂合并易错 | 高 | 严格单测与边界测试；参考成熟实现 |
| LOG-001 Redo 重放影响数据正确性 | 高 | 单独设计评审；崩溃恢复专项测试 |
| TXN-013 Next-Key 锁实现难度大 | 中 | 参考 InnoDB 文档与源码，分步实现 |
| 人力不足导致周期延长 | 中 | 优先保证 P0 与关键路径；P2 可裁剪 |

## 九、成功指标

### 9.1 功能指标

- [ ] 支持 READ COMMITTED、REPEATABLE READ
- [ ] 崩溃后可通过 Redo/Undo 恢复
- [ ] B+ 树完整 CRUD 与二级索引维护
- [ ] 子查询、谓词下推、列裁剪、连接顺序优化可用
- [ ] 预处理语句（COM_STMT_*）可用

### 9.2 性能指标（参考）

- [ ] 单表简单查询 QPS > 10,000
- [ ] 简单 JOIN QPS > 5,000
- [ ] 事务 TPS > 1,000
- [ ] 崩溃恢复时间 < 1 分钟（与数据量相关）

### 9.3 质量指标

- [ ] 核心路径单元测试覆盖
- [ ] 无 P0 未修复缺陷
- [ ] 关键设计有文档更新
